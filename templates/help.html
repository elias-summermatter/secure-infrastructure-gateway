Setup guide

← Back to dashboard

This gateway uses WireGuard — a small, fast VPN protocol — to securely tunnel only the traffic you need to reach protected infrastructure. Everything else (your normal internet) stays on your regular connection.

Your config file is a credential. The .conf file you download contains a private cryptographic key that identifies you personally. Treat it like a password. Never share it with anyone, never commit it to git, never paste it into chat. If it leaks, come straight back to the dashboard and click Rotate & download config — the old one is invalidated instantly.

How it works, in 30 seconds

  1. You download a personal WireGuard config (.conf) from the dashboard.
  2. You import it into the WireGuard client on your computer and activate the tunnel.
  3. On the dashboard, you activate a specific service (e.g. ssh) for up to 8 hours at a time.
  4. Your client now reaches that service's real hostname directly — certificates work normally, as if you were on the internal network.
  5. When your activation expires, the access goes away automatically.

Install WireGuard

Pick your operating system:

1. Install the WireGuard client

1

Download the installer

Get the official installer from wireguard.com/install (the link labelled Windows Installer).

2

Run the installer

Double-click the downloaded .msi file and accept the Windows admin prompt. Installation is silent and takes a few seconds.

2. Import your tunnel

1

Open WireGuard

Start it from the Start menu. The main window is empty the first time.

2

Import the .conf file you downloaded

Click Import tunnel(s) from file (bottom left) and pick the file (e.g. portcullis.conf) from your Downloads folder. You can also drag the file onto the window.

3

Activate the tunnel

Select the tunnel in the list and click Activate. Within a few seconds you'll see a green status and traffic counters start ticking.

Tip: Windows reliably auto-reconnects the tunnel after sleep/suspend. No extra configuration needed.

1. Install the WireGuard client

1

Install from the Mac App Store

Open the App Store, search for WireGuard, and install it (it's free and by WireGuard Development Team). Direct link: apps.apple.com/app/wireguard .

Homebrew alternative (advanced): 
brew install wireguard-tools
This gives you the CLI (wg-quick, wg) but no GUI; recommended only if you already use Homebrew heavily.

2. Import your tunnel

1

Open WireGuard

Launch it from Applications. A WireGuard icon appears in the menu bar.

2

Import the .conf file you downloaded

Click the menu-bar icon, then Manage Tunnels… → + (bottom left) → Import Tunnel(s) from File… and select your .conf.

3

Activate the tunnel

Toggle the switch next to the tunnel name. macOS will ask for permission to add a VPN configuration — click Allow. Approve it with Touch ID or your password.

Linux has two good paths. Pick the one that matches how you normally run your machine — both work equally well.

Option A — Graphical (GNOME, KDE, most desktop distros)

NetworkManager has built-in WireGuard support on any reasonably current distro (Ubuntu 22.04+, Fedora 36+, Debian 12+, Arch). You don't need to install anything extra.

1

Open your Network settings

GNOME: Settings → Network → click the + next to "VPN" → Import from file….
KDE Plasma: System Settings → Connections → + → Import VPN connection….

2

Pick the downloaded .conf

The settings will recognise it as WireGuard and pre-fill every field. Click Add / Save.

3

Toggle the connection on

From the same Network panel, flip the switch. You can also toggle from the system tray / status area menu like any other VPN.

Option B — Command line (servers, Sway, i3, or CLI-only machines)

Install the userspace tools (kernel support is built in):

# Debian / Ubuntu
sudo apt update && sudo apt install -y wireguard-tools

# Fedora / RHEL
sudo dnf install -y wireguard-tools

# Arch / Manjaro
sudo pacman -S wireguard-tools

Move the downloaded config into /etc/wireguard/ with strict permissions so wg-quick accepts it:

sudo install -m 600 ~/Downloads/portcullis.conf /etc/wireguard/

Bring the tunnel up:

sudo wg-quick up portcullis

Verify:

sudo wg show

You should see latest handshake: N seconds ago within ~5 seconds.

Tear it down when you're done:

sudo wg-quick down portcullis

Optional, auto-start on boot:

sudo systemctl enable wg-quick@portcullis

Using the gateway

Activate a service

Go back to the dashboard. You'll see a list of services configured for you. Click Activate on any one — you get access for 1 hour by default.

Extend an active session

Each active service shows a live countdown. Click Extend +1h to add another hour, capped at 8 hours remaining. You can keep clicking as time passes — there's no hard lifetime limit, you just never have more than 8 hours at once.

Deactivate early

Click Deactivate on any active service to end your session immediately. Any open TCP connections (SSH, psql, etc.) are killed within a second.

What happens when a timer expires

The gateway automatically tears down access. Your tunnel stays up, but it cannot reach any service until you activate one again. Open connections at the moment of expiry are also dropped.

Troubleshooting

"No handshake" / the tunnel doesn't connect

  • Your firewall may block outbound UDP on port 51820. Whitelist it.
  • If you're on a restrictive network (café, airport, hotel), WireGuard's UDP may be blocked entirely. You'd need a network change or a TCP-tunnel alternative (out of scope here).
  • Double-check the endpoint hostname/IP in your .conf resolves — from a terminal: ping <endpoint>.

Handshake works, but I can't reach the service

  • Did you click Activate on that service in the dashboard?
  • Has the activation timer expired? Check the countdown.
  • Is the service blocked for you by an admin? The service row will say "blocked by admin" and the Activate button will be disabled.
  • Did you just rotate your config? The old one is dead immediately — import the new one.

"Everything routes through the VPN and I'm slow"

This shouldn't happen — the config is scoped so only service IPs and the WG network (10.77.0.0/24) go through the tunnel. If you see otherwise, your AllowedIPs was modified; re-download a fresh config from the dashboard.

How do I stop the tunnel?

  • Windows / macOS: toggle the tunnel off in the WireGuard app.
  • Linux: sudo wg-quick down portcullis

Questions or problems?

Ask your infrastructure admin — they can see the audit log and help figure out what happened.
